# 루바토 톤앤매너

## 콘텐츠 프레임

planner는 주제에 맞는 프레임을 선택한다. 하나의 세트에 프레임을 섞지 않는다.

### 1. 인물 해석형

누군가의 말이나 행동에서 출발해 루바토의 시각으로 해석한다.

| 슬라이드 | 역할 |
|---------|------|
| 훅 | 인물의 말/행동 중 가장 강한 한 마디 |
| 인물 소개 | 누구인지, 왜 이 사람 이야기를 하는지 |
| 인용/핵심 주장 | 실제로 한 말 1~2문장 (출처 명시) |
| 해석 | "우리는 이걸 이렇게 본다" — 루바토의 관점 |
| 실용 연결 | 구체적으로 뭘 할 수 있는지 |
| 클로저 | 브랜드 마무리 |

예: 쇼피파이 CEO 인터뷰 → 핵심 발언 → 루바토가 보는 "기록의 의미" → 독서 노트 연결

### 2. 책 소개형

특정 책이나 작가를 소개하고, 읽을 이유와 기록을 연결한다.

| 슬라이드 | 역할 |
|---------|------|
| 훅 | 이 책에 대한 가장 강한 한 마디 (시의성, 의외성) |
| 책/작가 소개 | 맥락과 배경 |
| 핵심 내용 | 이 책이 말하는 것 1~2가지 |
| 데이터/비교 | 평점, 판매량, 다른 작품과 비교 등 |
| 감상 연결 | 읽고 나면 뭐가 달라지는지, 기록과 연결 |
| 클로저 | 브랜드 마무리 |

예: 프로젝트 헤일메리 영화 개봉 → 원작 3부작 소개 → 평점 비교 → "작품마다 감정이 다른 걸 적어두면 취향이 보인다"

### 3. 팁 리스트형

독자의 문제에서 출발해 구체적 방법을 제시하고 제품을 해결 수단으로 연결한다.

| 슬라이드 | 역할 |
|---------|------|
| 훅 | 독자의 고민/문제 (공감 유발) |
| 데이터/근거 | 왜 이게 문제인지 수치로 보여줌 |
| 전환 | "그래서 뭘 하면 되나?" |
| 팁 1~N | 구체적 방법 (번호 매기기) |
| 인사이트 | 핵심 메시지 요약 |
| 클로저 | 브랜드 마무리 |

예: "한 달 뒤 읽은 책의 79%가 사라진다" → 망각곡선 → 기록 방법 3가지 → 루바토 연결

### 4. 에세이형

오너 또는 루바토의 목소리로 경험과 성찰을 나눈다. 제품 연결은 최소화.

| 슬라이드 | 역할 |
|---------|------|
| 훅 | 경험에서 나온 한 마디 |
| 이야기 | 구체적 상황/경험 |
| 성찰 | 거기서 느낀 것 |
| 여운 | 독자에게 남기는 질문이나 문장 |
| 클로저 | 브랜드 마무리 (가볍게) |

예: "책을 덮자마자 적은 한 줄이 석 달 뒤에도 남았다" → 실제 경험 → 기록의 의미

### 5. 질문형

책에서 출발한 질문을 던져 독자의 생각과 대화를 유도한다.

| 슬라이드 | 역할 |
|---------|------|
| 훅 | 책에서 출발한 도발적 질문 |
| 책 소개 | 이 질문이 나온 맥락 |
| 질문 나열 | 생각해볼 질문 3~4개 |
| 관점 제시 | 루바토 또는 독자 후기 기반 해석 |
| 대화 유도 | "여러분은 어떻게 생각하세요?" |
| 클로저 | 브랜드 마무리 |

예: "지금 내가 하고 있는 생각은 진짜 나의 생각일까?" → 《거짓 공감》 → 토론 질문들

## 훅 소재 뱅크

planner가 훅을 만들 때 참고하는 소재. 프레임별로 다르게 활용한다.

| 훅 소재 | 적합한 프레임 |
|--------|-------------|
| "다 읽고 나면 기억나는 게 없다" | 팁 리스트형 |
| "독서 기록 노션으로 하다가 포기한 사람?" | 팁 리스트형 |
| "월 1권도 힘든데 독서 습관 만드는 법" | 팁 리스트형 |
| "끝까지 읽어야만 의미있을까" | 에세이형, 질문형 |
| "읽은 책 중에 다시 찾고 싶은 문장" | 팁 리스트형, 에세이형 |
| "올해 목표 50권인데 지금 몇 권이지" | 팁 리스트형 |
| "책 추천받고 바로 서재에 넣고 싶다" | 팁 리스트형 |
| 시의성 있는 영화/드라마 원작 | 책 소개형 |
| 유명 인물의 독서/기록 습관 | 인물 해석형 |
| 책에서 출발한 논쟁적 질문 | 질문형 |

## 톤앤매너 가이드

### 어조

- 친근하되 전문적 — "~이에요", "~해보세요" (반말 금지)
- 짧고 명확하게 — 한 슬라이드에 한 메시지
- 공감 우선 — 기능 설명 전 독자 상황 먼저

### 관점 원칙

루바토 공식 계정은 "독서 정보 전달자"가 아니라 "독서 기록에 대한 관점을 가진 브랜드"다.

- **큐레이션 근거를 드러낼 것** — "왜 이 주제를 다루는가"가 보여야 한다. dogfooding 경험, 자체 데이터, 구체적 출처를 활용. 검색하면 나오는 통계만으로 슬라이드를 채우지 않는다.
- **"루바토는 이렇게 본다"를 넣을 것** — 해석 슬라이드에서 브랜드 관점을 표명. 모든 독자가 동의 안 해도 된다. 관점이 있어야 기억에 남는다.
- **완벽한 정보보다 불완전한 솔직함** — 아쉬운 점, 한계, 예외도 언급 가능. "이 방법이 모든 사람에게 맞지는 않아요"가 "누구나 할 수 있어요"보다 신뢰를 준다.

### CTA 원칙

슬라이드 CTA는 광고가 아니라 대화다. 제품 CTA는 프로필 링크에 맡긴다.

- "루바토에서 시작하세요" (X) → "여러분은 어떤 기준으로 책을 고르세요?" (O)
- "팔로우하고 콘텐츠를 경험해보세요" (X) → "읽어본 책 있으면 알려주세요" (O)
- 질문형, 공유 유도형, 저장 유도형 CTA 사용

### 금지 표현

- 과도한 광고 어조 ("최고", "유일한", "완벽한")
- 긴 문장 (한 줄 35자 초과 지양)
- 영어 남용 (필요한 경우만 병기)

### 비주얼 원칙

- 다크↔라이트 슬라이드 교차 배치
- 숫자/강조 텍스트는 액센트 색상
- 여백을 충분히 (padding 최소 60px)
- 텍스트 최소 크기 28px (가독성)
- 세트별 액센트 컬러 변주 가능 (--accent 변수 활용)

## AI 트로프 회피 가이드

한국어 카드뉴스 카피에서 AI가 생성한 글임을 드러내는 패턴을 금지한다.

### 단어 선택

- "~의 세계로", "~의 여정" 같은 과장 표현 금지
- "활용하다", "극대화하다", "체계적으로" 등 AI 특유 딱딱한 표현 금지
- 간결하고 일상적인 단어 사용

### 문장 구조

- "X가 아니라 Y다" 부정 병렬 패턴 남발 금지 (가장 흔한 AI tell)
- "X도 아니다. Y도 아니다. 단지 Z일 뿐이다" 드라마틱 카운트다운 금지
- 3연속 반복 구조(tricolon) 남발 금지
- 수사적 질문 후 바로 답하기 패턴 금지

### 톤

- "사실은 간단합니다" 식 근거 없는 단정 금지
- "전문가들은 말합니다" 식 모호한 인용 금지
- "상상해보세요" 식 시나리오 빌딩 금지
- 교육적 톤("자, 하나씩 살펴볼까요?") 금지

### 포맷

- 엠대시(—) 과다 사용 금지
- 볼드 첫 단어 불릿 패턴 금지
- 모든 단락을 한 문장으로 만드는 패턴 금지
